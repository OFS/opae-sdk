\doxysubsection{Detailed Description}
Host/\+AFU shared memory blocks

\mbox{\hyperlink{classopae_1_1fpga_1_1types_1_1shared__buffer}{shared\+\_\+buffer}} abstracts a memory block that may be shared between the host cpu and an accelerator. The block may be allocated by the \mbox{\hyperlink{classopae_1_1fpga_1_1types_1_1shared__buffer}{shared\+\_\+buffer}} class itself (see allocate), or it may be allocated elsewhere and then attached to a \mbox{\hyperlink{classopae_1_1fpga_1_1types_1_1shared__buffer}{shared\+\_\+buffer}} object via attach.

Retrieve the virtual address of the buffer base.

\begin{DoxyNote}{Note}
Instances of a shared buffer can only be created using either \textquotesingle{}allocate\textquotesingle{} or \textquotesingle{}attach\textquotesingle{} static factory function. Because these functions return a shared pointer (std\+::shared\+\_\+ptr) to the instance, references to an instance are counted automatically by design of the shared\+\_\+ptr class. Calling \textquotesingle{}\mbox{\hyperlink{classopae_1_1fpga_1_1types_1_1shared__buffer_ade3a3102e5162ba8bb793342bdbf58f2}{c\+\_\+type()}}\textquotesingle{} function is provided to get access to the raw data but isn\textquotesingle{}t used in tracking its reference count. Assigning this to a variable should be done in limited scopes as this variable can be defined in an outer scope and may outlive the \mbox{\hyperlink{classopae_1_1fpga_1_1types_1_1shared__buffer}{shared\+\_\+buffer}} object. Once the reference count in the shared\+\_\+ptr reaches zero, the \mbox{\hyperlink{classopae_1_1fpga_1_1types_1_1shared__buffer}{shared\+\_\+buffer}} object will be released and deallocated, turning any variables assigned from a call to \textquotesingle{}\mbox{\hyperlink{classopae_1_1fpga_1_1types_1_1shared__buffer_ade3a3102e5162ba8bb793342bdbf58f2}{c\+\_\+type()}}\textquotesingle{} into dangling pointers. 
\end{DoxyNote}
